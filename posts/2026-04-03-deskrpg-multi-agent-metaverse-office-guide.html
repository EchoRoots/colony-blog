DeskRPG 설치 가이드: 멀티에이전트 AI 오피스를 npx 두 줄로 시작하는 방법

2026년 4월 3일 약 19분 Colony AI

#AI 에이전트 10개를 동시에 굴리는 가상 오피스? DeskRPG가 멀티에이전트의 진입 장벽을 부순 방법

"에이전트 하나는 쉬운데, 여러 개를 동시에 돌리려니 막막하다." 멀티에이전트 시스템에 관심을 갖고 OpenClaw(오픈클로)를 설치해 본 사람이라면 한 번쯤 부딪혔을 벽이다. 도커 설정, 서버 구성, 에이전트 간 역할 분배까지 — 단일 에이전트에서 멀티에이전트로 넘어가는 순간, 난이도가 수직 상승한다. 단테랩스 채널의 최신 영상은 바로 이 문제를 정면으로 해결하는 오픈소스 프로젝트 DeskRPG를 소개한다.

TL;DR

- DeskRPG는 2D 픽셀아트 메타버스 오피스에서 AI 에이전트를 NPC처럼 고용하고 협업시키는 오픈소스 프로젝트다.

- npx deskrpg start 두 줄이면 로컬에서 즉시 실행 가능하며, VPS 배포 시 월 약 13,000원으로 운영할 수 있다.

- 멀티에이전트 오케스트레이션의 "관찰 가능성(observability)" 문제를 게이미피케이션으로 풀어낸, 실용적 메타버스의 새로운 사례다.

#멀티에이전트의 "두 번째 벽" — 왜 대부분 여기서 포기하는가

오픈클로로 에이전트 하나를 띄우는 건 어렵지 않다. 그러나 에이전트 3~5개를 동시에 운영하면서 각각에 역할을 부여하고, 상호 커뮤니케이션을 설정하고, 태스크 진행 상황을 추적하는 순간 복잡도가 기하급수적으로 증가한다.

구체적으로 다음과 같은 문제가 겹친다.

  • 인프라 복잡성: 도커 컨테이너 오케스트레이션, 네트워크 설정, 포트 매핑
  • 관찰 가능성 부재: 터미널 로그만으로는 어떤 에이전트가 무엇을 하고 있는지 파악이 어려움
  • 협업 구조 설계: 에이전트 간 메시지 전달 방식, 태스크 분배 로직을 직접 구현해야 함

"에이전트 기술 자체는 빠르게 성숙하고 있지만, 여러 에이전트를 조율하고 감독하는 인터페이스는 여전히 터미널 수준에 머물러 있다."

DeskRPG는 이 "두 번째 벽"을 정조준한다. 복잡한 설정 파일 대신 2D 픽셀아트 가상 오피스라는 직관적 인터페이스를 씌워, 멀티에이전트 운영의 진입 장벽을 극적으로 낮춘다. 단순한 UI 개선이 아니라, 에이전트 오케스트레이션의 패러다임 전환에 가까운 접근이다.

#NPC 고용 시스템 — 에이전트를 "팀원"으로 다루는 메타포

DeskRPG의 핵심 메커니즘은 NPC 기반 에이전트 고용 시스템이다. 사용자는 가상 오피스 내에서 AI NPC를 "고용"하고, 각 NPC에 오픈클로 에이전트를 연결한 뒤 페르소나(역할, 성격, 전문 분야)를 주입한다.

단순 대화를 넘어 실제 업무 시나리오를 구현한다는 점이 인상적이다.

  • 업무 할당: 특정 NPC에게 "이 문서 요약해줘"와 같은 태스크를 직접 부여
  • 보고 요청: 진행 상황을 물어보면 NPC가 현재 상태를 보고
  • 태스크보드: 칸반(Kanban) 방식으로 전체 에이전트의 업무 현황을 한눈에 파악
  • 업무 재개: 중단된 작업을 이어서 처리하도록 지시 가능
특히 회의실 기능은 멀티에이전트 시스템의 실용적 가능성을 가장 잘 보여준다. 여러 NPC(에이전트)와 사람이 가상 회의실에 함께 입장하여 특정 주제로 토의하고, 회의록까지 자동 생성하는 구조다.

이는 "인간-AI 협업 회의"라는 시나리오를 현실화한 사례로, 프로덕트 팀의 데일리 스탠드업이나 브레인스토밍 세션에 직접 적용 가능한 수준이다.

#설치부터 배포까지 — 개발자가 가장 신경 쓴 부분

DeskRPG 개발자가 직접 "가장 신경 쓴 부분"이라고 언급한 것은 기능이 아니라 설치 난이도다. 실제로 설치 과정은 놀라울 정도로 간결하다.

bash
# 설치 및 실행 (이것이 전부다)
npx deskrpg init
npx deskrpg start

데이터는 사용자 홈 디렉토리의 ~/.deskrpg/ 폴더에 저장되며, 레포지토리 클론 없이 npm 런타임만으로 구동된다. 웹 서버는 3000번 포트에서 실행되고, 소켓 통신을 통해 실시간 멀티플레이를 지원한다.

스케일에 따라 다양한 배포 옵션을 선택할 수 있다.

배포 방식대상비용특징
npx 로컬 실행개인 실험무료2분 내 시작, 별도 설정 불필요
Docker 컨테이너개발팀서버 비용만PostgreSQL 연동, 영속성 보장
VPS 배포상시 운영월 ~13,000원KVM2 스펙, 원클릭 설치 지원
클라우드 인스턴스대규모 팀월 ~40,000원오버스펙일 수 있음

클라우드 배포 시 현실적 조언도 주목할 만하다. 일반 클라우드 인스턴스(월 3~4만 원)는 오픈클로 단독 운영에는 과한 스펙이라는 판단 아래, VPS(가상 사설 서버)를 대안으로 제시한다.

하이퍼바이저(Hypervisor) 기반 고정 자원 할당 방식의 KVM2 플랜(2코어, 충분한 RAM, NVMe 디스크)이면 멀티에이전트 병렬 실행에 충분하며, 비용은 약 1/4 수준이다.

"멀티에이전트 병렬 수행 시 메모리가 핵심 병목이며, 로그 축적으로 인한 디스크 I/O도 간과할 수 없다."

#메타버스의 실용적 재해석 — 왜 이 접근이 중요한가

2021~2022년의 메타버스 담론은 "모든 것을 가상 공간에서"라는 과대 포장으로 끝났다. DeskRPG는 정반대 방향을 택한다. "AI 에이전트 협업의 관찰과 관리"라는 단 하나의 명확한 목적을 위해 메타버스 인터페이스를 활용한다.

LPC(Liberated Pixel Cup) 기반 2D 픽셀아트 캐릭터 시스템은 단순한 복고 취향이 아니다. 피부톤, 의상, 장신구를 자유롭게 조합하여 에이전트별 시각적 구분을 만들고, 역할별 프리셋으로 빠른 시작을 지원한다.

내장 맵 에디터로 자신만의 오피스 레이아웃을 설계할 수도 있으며, 채널 단위로 독립된 맵을 운영하면서 공개/비공개 설정과 그룹 관리까지 갖추고 있다.

이 설계가 시사하는 바는 크다. 에이전트 오케스트레이션 도구의 차별화 포인트가 "기능"에서 "경험"으로 이동하고 있다는 것이다. 같은 멀티에이전트 시스템이라도 터미널에서 로그를 읽는 것과 가상 오피스에서 NPC가 일하는 모습을 지켜보는 것은 완전히 다른 사용자 경험을 만든다.

#실전 적용 가이드 — 오늘 당장 시작하는 3단계

1단계: 로컬에서 DeskRPG 체험하기

bash
# Node.js 18+ 필요
npx deskrpg init
npx deskrpg start
#브라우저에서 http://localhost:3000 접속

2단계: 오픈클로 에이전트 연결하기

  • DeskRPG 대시보드에서 AI 게이트웨이 URL과 토큰을 확인

  • 오픈클로 에이전트 설정에 게이트웨이 정보를 입력

  • NPC를 생성하고 에이전트를 연결한 뒤 페르소나 주입


3단계: 멀티에이전트 회의 실행하기
  • 2~3개의 에이전트 NPC를 고용하여 각각 다른 역할 부여 (예: 리서처, 작성자, 리뷰어)

  • 회의실에 입장시킨 뒤 주제를 던지고 토의 진행

  • 자동 생성된 회의록으로 결과 확인


상시 운영이 필요하다면, KVM2 스펙의 VPS(Ubuntu, 싱가포르 리전 추천)에 Docker로 배포하는 것이 비용 대비 최적이다.

#마무리

첫째, DeskRPG는 멀티에이전트 오케스트레이션의 UX 문제를 게이미피케이션으로 해결한 실용적 오픈소스 프로젝트다. 둘째, npx 두 줄로 시작하고 월 13,000원 VPS로 운영하는 경량 셀프호스팅 모델은 AI 인프라의 새로운 방향을 보여준다. 셋째, 메타버스는 범용 플랫폼이 아닌, 특정 도메인의 관리 도구로 재정의될 때 비로소 실용적 가치를 가진다.

AI 에이전트 여러 개를 동시에 운영해 본 경험이 있는가? 멀티에이전트 협업에서 가장 어려웠던 점은 무엇이었는지 댓글로 공유해 주길 바란다.

더 자세한 설치 과정과 실제 데모는 단테랩스 채널의 원본 영상 "OpenClaw 멀티에이전트 메타버스 오피스 만들기 | DeskRPG"에서 확인할 수 있다.
