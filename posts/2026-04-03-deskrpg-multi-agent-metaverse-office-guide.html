#AI 에이전트 10개를 동시에 굴리는 가상 오피스가 있다면? DeskRPG가 제시하는 멀티에이전트 오케스트레이션의 미래

#도입부

AI 에이전트 하나는 누구나 돌린다. 문제는 둘 이상부터다. 도커 설정, 네트워크 구성, 역할 분배, 상태 모니터링까지 — 에이전트 수가 늘어날수록 복잡성은 기하급수적으로 증가한다. 이 진입 장벽을 2D 픽셀아트 메타버스로 허물겠다는 오픈소스 프로젝트가 등장했다. 단테랩스 채널의 "OpenClaw 멀티에이전트 메타버스 오피스 만들기" 영상에서 소개된 DeskRPG를 심층 분석한다.

TL;DR

- DeskRPG는 오픈클로(OpenClaw) 멀티에이전트를 RPG 감성의 가상 오피스에서 시각적으로 운영할 수 있는 오픈소스 도구다.

- npx 명령어 두 줄이면 설치가 끝나며, SQLite부터 Docker/VPS 배포까지 유연하게 확장된다.

- NPC로 에이전트를 고용하고, AI 회의를 열어 회의록까지 자동 생성하는 실무형 멀티에이전트 인터페이스를 제공한다.

#왜 멀티에이전트에는 새로운 인터페이스가 필요한가

오픈클로로 단일 에이전트를 운영하는 것은 비교적 간단하다. 하지만 에이전트를 3개, 5개, 10개로 확장하는 순간 상황이 달라진다. 각 에이전트의 역할 정의, 상호 통신, 상태 모니터링, 장애 대응까지 — 터미널 로그만으로는 관찰 가능성(Observability)을 확보하기 어렵다.

기존 멀티에이전트 도구들은 대부분 YAML 설정 파일이나 CLI 기반 인터페이스에 의존한다. 개발자에게는 익숙하지만, 에이전트 수가 늘어날수록 전체 시스템의 상태를 한눈에 파악하기 어렵다는 구조적 한계가 있다.

DeskRPG는 이 문제를 공간적 메타포로 풀었다. 에이전트가 어디서 무엇을 하고 있는지, 누가 누구와 대화 중인지를 2D 맵 위에서 직관적으로 확인할 수 있다. 단순한 UI 개선이 아니라, 멀티에이전트 오케스트레이션의 UX를 근본적으로 재정의하려는 시도다.

#NPC를 고용하라: 게이미피케이션이 만든 직관적 에이전트 관리

DeskRPG의 핵심 설계 원칙은 "NPC = AI 에이전트"라는 메타포다. 사용자는 오픈클로 에이전트를 가상 오피스의 NPC로 "고용"하고, 페르소나를 주입하여 특정 업무를 맡긴다.

이 구조가 강력한 이유는 복잡한 오케스트레이션 개념을 게임적 경험으로 치환했기 때문이다:

  • 업무 위임: NPC에게 말을 걸어 태스크를 할당
  • 진행 관리: 태스크보드를 통해 각 에이전트의 작업 상태 추적
  • 중단/재개: 에이전트의 작업을 언제든 중단하고 다시 시작
  • 보고 요청: 완료된 작업에 대한 결과 보고를 자연어로 요청
특히 주목할 점은 멀티플레이어 협업과 AI 회의 시스템이다. 전용 회의실에서 복수의 NPC 에이전트가 특정 주제로 토의하고, 사람과 AI가 함께 참여하는 하이브리드 회의를 진행할 수 있다. 회의록 자동 생성까지 지원된다.

단일 에이전트의 지시-응답 패턴을 넘어, "회의하는 AI"라는 새로운 패러다임을 구체화한 사례다.

#설치 2분, 배포는 원하는 대로: 유연한 인프라 설계

셀프호스팅 도구의 최대 적은 설치 포기율이다. DeskRPG 개발자가 가장 공을 들인 부분도 바로 이 지점이다.

bash
npx deskrpg        # 설치
npx deskrpg start  # 실행

Node.js만 설치되어 있으면 이 두 줄로 끝난다. 모든 데이터는 ~/.deskrpg/ 디렉토리에 저장되며, 기본 DB는 SQLite로 별도 설정이 불필요하다.

프로덕션 환경으로의 확장도 단계적으로 설계되어 있다:

단계구성적합한 상황
로컬SQLite + localhost:3000개인 프로토타이핑
서버PostgreSQL + WebSocket팀 단위 상시 운영
Docker오픈클로 통합 컨테이너멀티에이전트 병렬 운영
VPSKVM2 + NVMe SSD비용 최적화 프로덕션

비용 측면에서도 현실적인 대안을 제시한다. 일반 클라우드 인스턴스(월 3~4만 원)는 단일 에이전트 서버에 오버스펙이라는 판단 하에, VPS(월 약 1.3만 원)를 권장한다. 호스팅어의 Docker Manager를 활용하면 원클릭 배포도 가능하며, 서버 위치는 싱가포르가 권장된다.

권한 체계도 실무를 고려했다. 최초 접속자가 관리자 권한을 갖고, 이후 접속자에게 역할을 부여하는 구조다. 채널(맵) 단위로 공개/비공개 설정이 가능하여 팀별 독립 운영이 된다.

#실전 적용 가이드: 오늘 당장 시작하는 4단계

Step 1 — 로컬에서 체험하기
Node.js 설치 후 npx deskrpg && npx deskrpg start를 실행한다. localhost:3000에서 가상 오피스에 접속할 수 있다.

Step 2 — 오픈클로 연동
오픈클로 대시보드 URL과 게이트웨이 토큰을 DeskRPG 설정에 입력한다. 이후 NPC 에이전트를 생성하고 페르소나를 부여한다.

Step 3 — 맵 커스터마이징
기본 템플릿으로 시작하되, 내장 맵 에디터로 팀 구조에 맞는 오피스를 설계한다. LPC(Liberated Pixel Cup) 기반 캐릭터 시스템으로 아바타도 커스터마이징 가능하다.

Step 4 — VPS 배포로 상시 운영 전환
팀 규모가 확대되면 KVM2 스펙의 VPS에 Docker로 배포한다. 멀티에이전트 병렬 수행 시 RAM이 핵심 병목이므로 메모리 여유를 확보하고, 로그 적재를 위해 NVMe SSD를 선택한다.

#마무리: 에이전트 관리의 다음 장

멀티에이전트 시대의 진짜 병목은 AI의 성능이 아니라 운영 인터페이스다. DeskRPG는 게이미피케이션으로 이 문제를 정면 돌파한 초기 사례이며, "시각적 에이전트 오케스트레이션"이라는 새로운 카테고리의 가능성을 열었다.

셀프호스팅 생태계가 AI 에이전트 영역으로 확장되면서, 에이전트는 API 호출이 아닌 상시 운영 서비스로 진화하고 있다. 여러분의 팀에서 멀티에이전트를 도입한다면, 어떤 인터페이스가 가장 효과적일까?

더 자세한 설치 과정과 실제 운영 데모는 단테랩스의 원본 영상에서 확인할 수 있다.
